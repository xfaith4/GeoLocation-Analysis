# Contributing to GeoLocation Analysis

Thank you for your interest in contributing to the GeoLocation Analysis project!

## Development Setup

1. Fork the repository
2. Clone your fork
3. Create a virtual environment: `python3 -m venv venv`
4. Activate it: `source venv/bin/activate`
5. Install dependencies: `pip install -r requirements.txt`

## Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep functions small and focused

## Testing

Before submitting a PR:

1. Test your changes locally
2. Ensure the web application runs without errors
3. Test with various NMEA file formats if changing the parser

## Submitting Changes

1. Create a new branch for your feature
2. Make your changes
3. Test thoroughly
4. Commit with clear, descriptive messages
5. Push to your fork
6. Open a Pull Request

## Reporting Issues

- Use the GitHub issue tracker
- Provide detailed information about the bug or feature request
- Include sample GNSS data if relevant (ensure it doesn't contain sensitive location information)

## Questions?

Feel free to open an issue for any questions about contributing.
